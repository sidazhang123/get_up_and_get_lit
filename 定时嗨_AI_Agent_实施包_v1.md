## 0. 给 AI agent 的执行指令

你需要实现一个 Android 原生 App「定时嗨」。  
请严格按本文件实施，不要自行扩展为“长期日历提醒”或“每日重复提醒”。本产品是**单批次、多任务、一次性预定执行**模型。  

执行要求：

1. 不要擅自改变页面文案、状态枚举、按钮命名、显隐规则。
2. 不要把任务语义改造成每日循环。
3. 手机重启后，不做恢复、不做补触发、不做失败补偿。
4. UI 必须从数据库状态驱动；后台服务只写数据库，不直接操作页面。
5. 若实现过程中发现未定义细节，优先选择与本文件现有规则一致、最保守、最少新增能力的实现方式。
6. 交付物至少包括：可运行工程、核心说明、关键日志点、基础自测结果。

## 1. 产品目标与非目标

### 1.1 产品目标
“定时嗨”用于在本机预先配置若干音频播放任务，并在用户手动点击“全部预定”后，以一轮批次的方式在后续时间点依次执行。  
每个任务在同一批次内最多执行一次。产品核心能力是：

- 创建、编辑、删除本机定时播放任务
- 发起和取消一轮批次调度
- 在后台/锁屏场景下尽可能稳定执行
- 蓝牙不可用时终止任务并记录状态、log
- 新任务到点时可抢占当前播放任务

### 1.2 非目标
以下能力不在本次实现范围内：

- 不支持按周、按月、按日期等长期重复规则
- 不支持云同步、多设备同步、服务端调度
- 不支持在线播放资源，仅支持本地 mp3 / wav
- 不支持手机重启后的任务恢复、错过补偿、失败补偿
- 不支持同一 HH:MM 的多个任务共存

## 3. 核心业务模型

### 3.1 批次模型
本产品不是“任务创建后自动永久生效”，而是“用户点一次全部预定，触发一轮批次执行”。

规则如下：

1. 用户点击“全部预定”后，进入一轮批次。
2. 批次内，每个任务最多执行一次。
3. 所有任务状态均不再是“未触发”时，scheduler 自动退出。
4. 若用户要再次执行全部任务，必须再次点击“全部预定”。
5. 点击“取消预定”后，本轮批次立即停止。
6. 手机重启后，停止所有任务和 scheduler；所有任务状态统一重置为“未触发”。

### 3.2 任务触发时间模型
任务输入的是 `HH:MM`，真正触发时点按 `HH:MM:00` 处理。

对本轮批次中每个任务：

- 若批次启动时，当日 `HH:MM:00` 尚未到达，则目标触发时刻 = 当日 `HH:MM:00`
- 若批次启动时，当日 `HH:MM:00` 已过去，则目标触发时刻 = 次日 `HH:MM:00`

注意：  
这里的“次日”只是为了完成本轮批次中的一次执行，不代表该任务是“每日重复任务”。

## 4. 状态定义

### 4.1 任务状态
任务状态只允许以下 3 个值：

- `未触发`
- `已触发`
- `蓝牙未连接`

不得新增“执行中”“已错过”“失败”“已取消”等前台持久状态。  
播放失败、文件失效、用户取消、系统事件等可写 log，但不要扩展前台状态枚举。

### 4.2 全局调度状态
全局只区分以下 2 个状态：

- `未预定`
- `批次执行中`

映射规则：

- 首页右上按钮显示“全部预定”时，对应 `未预定`
- 首页右上按钮显示“取消预定”时，对应 `批次执行中`

## 5. 数据模型

建议使用单表 `tasks`。

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| id | Long | 是 | 主键，自增 |
| name | String | 否 | 任务名称；可为空；最大 10 字 |
| startHour | Int | 是 | 0-23 |
| startMinute | Int | 是 | 0-59 |
| fileUri | String | 是 | 选择文件后的 Uri 字符串 |
| fileName | String | 是 | 展示用文件名 |
| playMode | String | 是 | `single` / `interval` |
| loopCount | Int? | 否 | 仅 `interval` 模式需要；范围 1-99 |
| intervalMinSec | Int? | 否 | 仅 `interval` 模式需要；范围 1-9999 |
| intervalMaxSec | Int? | 否 | 仅 `interval` 模式需要；范围 1-9999，且 >= min |
| status | String | 是 | `未触发` / `已触发` / `蓝牙未连接` |
| createdAt | Long | 是 | 创建时间 |
| updatedAt | Long | 是 | 更新时间 |

### 5.1 数据约束
1. 新建任务时，`startHour + startMinute` 组合不得与任何已有任务完全相同。
2. 编辑任务时，允许保留当前任务原有 `HH:MM`；但若修改后与其他任务重复，则保存失败。
3. `playMode = single` 时，`loopCount / intervalMinSec / intervalMaxSec` 必须为空。
4. `playMode = interval` 时，上述 3 个参数必须完整存在并通过校验。
5. 所有页面展示均基于数据库读取结果。

## 6. 页面规格

### 6.1 主页
页面元素：

- 顶部左侧文字按钮：`创建任务`
- 顶部右侧按钮：
  - 未预定时：`全部预定`
  - 批次执行中时：`取消预定`
- 中部：任务列表
- 支持下拉刷新

列表项展示固定三行：

1. `任务名称 + 开始时间`
2. `执行状态：xxx`
3. `播放安排：单次播放 / 间隔循环播放`

交互规则：

1. 未预定时：
   - 可点击“创建任务”
   - 列表可左右滑
   - 可下拉刷新
2. 批次执行中时：
   - “创建任务”按钮禁用
   - 列表锁定，不允许左滑删除、右滑编辑
   - 仍可查看列表与状态变化
3. 左滑列表项：删除
4. 右滑列表项：编辑
5. 删除时需要二次确认

### 6.2 任务创建页
顶部标题必须为：`任务创建页`

字段与规则：

1. 任务名称  
   - 非必填
   - 最大 10 字

2. 任务开始时间  
   - 必填
   - 输入为 `HH:MM`
   - 小时范围 0-23
   - 分钟范围 0-59

3. 播放文件  
   - 必填
   - 仅支持 `mp3` / `wav`

4. 任务安排  
   - 必选
   - `单次播放`
   - `间隔循环播放`

显隐规则：

- 当未选中 `间隔循环播放` 时，不显示下方参数区，也不显示任何占位说明文字
- 当选中 `间隔循环播放` 时，显示以下参数：
  - 循环次数：1-99
  - 每次循环中间隔最小值：1-9999 秒
  - 每次循环中间隔最大值：1-9999 秒，且必须 >= 最小值

页面动作：

- 左上：`取消`
- 右上：`保存`

保存规则：

- 任一必填项缺失 -> 阻止保存并提示
- 校验失败 -> 阻止保存并提示
- 保存成功 -> 返回主页并刷新列表

取消规则：

- 不保存任何变更
- 直接返回主页

### 6.3 任务编辑页
任务编辑页复用任务创建页布局，但顶部标题必须改为：`任务编辑页`

其他规则：

- 字段初始值加载当前任务内容
- 编辑保存时沿用创建页全部校验规则
- 编辑页也遵守“间隔循环播放参数区按选择显隐”的规则

## 7. 事件规则

### 7.1 点击“全部预定”
按以下顺序执行：

1. 若存在当前播放中的任务，先停止播放
2. 若存在正在运行的 scheduler，先停止 scheduler
3. 将所有任务状态统一更新为 `未触发`
4. 计算当前批次所有任务的目标触发时刻
5. 启动前台服务与 scheduler
6. 首页进入 `批次执行中`
7. 按钮文案切为 `取消预定`
8. 禁用“创建任务”和列表侧滑编辑/删除

### 7.2 点击“取消预定”
按以下顺序执行：

1. 停止当前播放任务（若存在）
2. 停止 scheduler
3. 首页回到 `未预定`
4. 按钮文案切回 `全部预定`
5. 恢复“创建任务”和列表侧滑编辑/删除能力

说明：  
点击“取消预定”时，不要求把已有 `已触发` / `蓝牙未连接` 状态重新改回 `未触发`。  
只有点击“全部预定”时，才统一重置所有任务状态为 `未触发`。

### 7.3 新任务到点时
1. 若已有任务正在播放，先终止旧任务
2. 对当前到点任务先执行蓝牙检查 wrapper
3. 若蓝牙不可用：
   - 将任务状态置为 `蓝牙未连接`
   - 写 log
   - 终止本轮任务
4. 若蓝牙可用：
   - 先将任务状态置为 `已触发`
   - 进入播放流程

### 7.4 手机重启
手机重启后：

1. 不恢复上一轮批次
2. 不补触发错过任务
3. 不补记失败任务
4. 停止所有任务和 scheduler
5. 所有任务状态统一改为 `未触发`
6. 首页进入 `未预定`

## 8. 调度器规格

### 8.1 调度器所有权
- scheduler 必须只有一个真实运行实例
- scheduler 的唯一所有者应为前台服务
- 页面层不得自建并持有独立 scheduler

### 8.2 调度器退出条件
当所有任务状态都不是 `未触发` 时，scheduler 自动退出，并将全局状态切回 `未预定`

### 8.3 调度器伪代码
```pseudo
onReserveAll():
    stopCurrentPlaybackIfAny()
    stopSchedulerIfAny()
    resetAllTaskStatusToUntriggered()
    batchTargets = computeTargetDateTimes(now, allTasks)
    startForegroundService()
    scheduler.start(batchTargets)

scheduler.loop():
    while active:
        if all task.status != "未触发":
            stopScheduler()
            setGlobalState("未预定")
            break

        nextTask = findNearestUntriggeredTask()
        sleepUntil(nextTask.targetDateTime)

        if scheduler not active:
            break

        handleTaskTrigger(nextTask)

handleTaskTrigger(task):
    stopCurrentPlaybackIfAny()

    if bluetoothWrapper() == false:
        update task.status = "蓝牙未连接"
        writeLog(task, "bluetooth_unavailable")
        return

    update task.status = "已触发"
    playTask(task)
```

### 8.4 同时到点冲突
虽然业务约束已禁止同一 `HH:MM` 的任务并存，但实现层仍建议保留保护：

- 若发现多个未触发任务目标时刻完全一致，按 `updatedAt` 降序取第一条执行，其余写 log 并跳过  
- 该保护仅作兜底，不作为正常产品能力

## 9. 播放规则

### 9.1 单次播放
- 后台静默播放所选文件一次
- 不拉起任何 Activity
- 不切前台页面
- 正常输出音频，不是“静音播放”

### 9.2 间隔循环播放
规则如下：

1. 第一次立即播放，并计入循环次数
2. 总播放次数 = 用户填写的循环次数 `N`
3. 每次播放结束后，从 `[intervalMinSec, intervalMaxSec]` 闭区间随机生成整数秒数
4. 等待上述秒数后再次播放
5. 达到 `N` 次后结束该任务

### 9.3 播放异常
若文件读取失败、Uri 无权限、播放器启动失败、播放中异常终止等：

- 本轮任务立即终止
- 写 log
- 不自动重试
- 不新增前台任务状态
- 不影响其他后续任务继续按调度执行

## 10. 蓝牙检查 wrapper

触发任务前必须执行 wrapper。  
wrapper 的判断标准：

- 当前存在可用的蓝牙音频输出设备
- 系统音频路由可切到该设备

若返回 `false`：

- 任务状态 -> `蓝牙未连接`
- 写 log
- 当前任务结束

若返回 `true`：

- 进入播放流程

注意：  
“蓝牙未连接”是当前批次下该任务的最终结果之一。  
后续即使蓝牙重新连接，也不要自动重试该任务。  
若用户想重新执行，必须再次点击“全部预定”。

- 写 log
- 不自动重试
- 不新增前台状态

## 15. 验收用例

18. 播放异常时任务终止并写 log
19. 上述失败不会阻断后续任务继续执行

### 15.4 重启
20. 手机重启后，所有任务状态统一回到 `未触发`
21. 手机重启后，scheduler 不恢复
22. 手机重启后，首页为 `未预定`
23. 手机重启后，不补触发任何错过任务
